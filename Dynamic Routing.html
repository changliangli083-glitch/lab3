The Problem Statement

"Current ISP routing is not suitable to changing consumer preferences and is failing to adapt to new internet usage trends.
In heterogeneous network environments, the coexistence of diverse traffic types such as UDP and TCP presents unique challenges for routing and resource allocation. UDP traffic is highly sensitive to latency and jitter, requiring stable and responsive routing paths to maintain real-time performance. Conversely, TCP traffic demands reliable throughput and benefits from bandwidth-optimized, multi-path routing strategies. Traditional static routing approaches often fail to accommodate these distinct requirements, particularly in multi-user settings where demand fluctuates dynamically. To address this, a protocol-aware and user-centric routing system is proposed, capable of real-time traffic classification, link-state monitoring, and priority-based path selection. This system enhances both network efficiency and user experience by dynamically adjusting routing behavior according to flow characteristics and user criticality."

The Context

The project addresses the limitations of traditional, topology-driven routing methods that lack awareness of application-specific requirements like latency, security, and quality of service. It proposes an application-aware routing framework using Software-Defined Networking (SDN) that can dynamically direct traffic based on the nature of the application. The system is built on Mininet and POX, utilizing flow classification to apply service-based routing policies. Evaluation involves simulating various service types and analyzing traffic behavior and controller performance. The ultimate goal is to enable intent-based routing strategies suitable for multi-tenant and SLA-sensitive environments.
